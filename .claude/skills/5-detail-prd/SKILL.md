# 5. Detail PRD

1번에서 정한 프로젝트 방향을 상세 PRD(Product Requirements Document)로 발전시키는 단계입니다.

## When to Use

- 사용자가 `/5-detail-prd` 을 입력했을 때

## Instructions

### Step 0 (조용히): 추적 시작

```bash
bash scripts/track.sh update 5-detail-prd started
```

> 이 단계는 사용자에게 보여주지 않습니다. `.fearnot/participant.json`이 없으면 자동 스킵됩니다.

## Step 5: 상세 기획서 (PRD) 작성

```
1번에서 만든 설계도            상세 기획서 (PRD)
┌──────────────────┐       ┌──────────────────────────┐
│ 핵심기능: 주문     │       │ 주문 기능:                │
│        결제      │       │  - 메뉴 선택 → 장바구니    │
│        알림      │  ──→   │  - 수량 변경 가능          │
│                  │  상세화 │  - 최소 주문금액 10,000원  │
│  (뼈대만 있음)     │       │  - 결제 후 주문번호 발급   │
└──────────────────┘       │  - 예상 배달시간 표시      │
                           └──────────────────────────┘
```

**쉽게 말하면:**
1번에서 "이런 집을 짓겠다"는 큰 그림을 그렸다면,
이번엔 "거실은 몇 평, 창문은 어디에, 콘센트는 몇 개" 같은
**세부 사항**을 정하는 거예요.

이게 있어야 Claude가 "아, 이 버튼을 누르면 이렇게 동작해야 하는구나"
하고 정확하게 코드를 짤 수 있어요.

### Step 3: CLAUDE.md 읽기

프로젝트의 CLAUDE.md를 읽어서 1번에서 정의한 핵심 기능 목록을 파악합니다.

### Step 4: clarify:vague로 각 기능 상세화

**clarify:vague skill을 호출**하되, 이번에는 핵심 기능 하나하나를 상세화하는 데 집중합니다.

각 핵심 기능에 대해 아래를 정리:
- **사용자 스토리**: "~로서, ~을 하고 싶다, ~을 위해"
- **상세 동작**: 버튼을 누르면 어떻게 되는지 단계별
- **화면 구성**: 어떤 화면이 필요한지, 각 화면에 뭐가 있는지
- **데이터**: 어떤 데이터를 저장/표시하는지
- **예외 처리**: 에러가 나면 어떻게 할지
- **우선순위**: Must-have / Nice-to-have

### Step 5: PRD 문서 생성

프로젝트 폴더에 `docs/PRD.md`를 생성합니다.

```bash
mkdir -p projects/{프로젝트명}/docs
```

아래 구조로 PRD를 작성:

```markdown
# {프로젝트명} - PRD (Product Requirements Document)

## 1. 프로젝트 개요
{CLAUDE.md에서 가져온 개요}

## 2. 사용자 페르소나
{대상 사용자를 구체적인 인물로 묘사}

## 3. 핵심 기능 상세

### 3.1 {기능 1}
**사용자 스토리**: {~로서, ~을 하고 싶다}
**화면**: {필요한 화면들}
**동작 흐름**:
1. {step 1}
2. {step 2}
3. ...
**데이터 모델**: {필요한 데이터}
**예외 처리**: {에러 시나리오}
**우선순위**: Must-have / Nice-to-have

### 3.2 {기능 2}
...

## 4. 화면 목록
| # | 화면명 | 경로 | 설명 |
|---|--------|------|------|
| 1 | 홈 | / | ... |
| 2 | 로그인 | /login | ... |

## 5. 데이터 모델 초안
| 모델 | 주요 필드 | 관계 |
|------|----------|------|
| User | email, name, ... | has many Orders |

## 6. API 엔드포인트 초안
| Method | Path | 설명 |
|--------|------|------|
| GET | /api/... | ... |

## 7. 구현 우선순위
### Phase 1 (MVP)
- [ ] {기능}
### Phase 2
- [ ] {기능}
```

### Step 6: CLAUDE.md 업데이트

PRD 내용 중 핵심 사항을 CLAUDE.md에도 반영합니다.
특히 화면 목록, 데이터 모델, API 구조를 CLAUDE.md에 추가하여
Claude가 코드 작성 시 바로 참고할 수 있게 합니다.

완료 추적 (조용히):

bash scripts/track.sh update 5-detail-prd completed
