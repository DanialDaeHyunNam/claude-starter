# Find Context

현재 대화/작업과 관련된 **과거 맥락**을 프로젝트 문서에서 찾아 정리해주는 유틸리티 skill.

## Usage

- `/find-context` — 최근 대화 흐름에서 주제 추론
- `/find-context {주제}` — 주제를 직접 지정 (예: `/find-context 결제 플로우`)

## When to Use

- 사용자가 `/find-context` 를 입력했을 때
- 사용자가 "예전에 왜 이렇게 했더라", "과거 결정 확인", "관련 맥락 찾아줘" 라고 물을 때
- 새로운 작업을 시작하기 전, 관련 기록이 있는지 확인이 필요할 때

## Instructions

이 skill은 **유틸리티 도구**입니다. 다른 단계 skill과 달리 "설명 → 이해 확인" 패턴 없이 바로 진단을 실행합니다.

### Step 1: 주제(topic) 확정

우선순위:

1. **인자로 주제가 들어왔으면** 그것을 주제로 사용
2. **없으면** 최근 대화에서 중심 주제의 키워드를 2~5개 추출
   - 예: "결제 플로우", "인증 방식", "프로필 화면 구조"
3. **주제가 모호하면** AskUserQuestion 한 번만 물어봐서 확정
   - question: "어떤 주제에 대한 과거 맥락을 찾을까요?"
   - options: 최근 대화에서 추출한 후보 주제 2~4개 + "직접 입력"

### Step 2: 검색 대상 탐색

존재하는 경로만 검색 대상으로 삼는다. 없으면 건너뛴다.

```
docs/prd/**/*.md              ← 상세 PRD
docs/**/*.md                  ← 기타 기획/설계 문서
CLAUDE.md                     ← 프로젝트 설계도
.omniscitus/**/*.md           ← wrap-up / history units (omniscitus 사용 시)
.claude/member/*/done/**/*.md ← 완료된 작업 단위 (팀 기반 워크플로우 시)
_weekly/**/*.md               ← 주간 요약
```

Glob으로 존재 여부만 빠르게 확인한 뒤, 존재하는 경로에서만 Grep을 수행한다.

### Step 3: 신호 필터링 (단순 매치가 아닌 'why' / 'lessons learned' 추출)

단순히 키워드만 찾지 말고, **의미 있는 신호**가 섞인 부분을 골라낸다.

**"Why" (의사결정) 신호:**
- 결정, 선택, 이유, 왜냐하면, 때문에, 대신, rationale, decided, why
- ~로 정했다, ~를 선택한 이유, Trade-off, 대안으로는, 고려한 선택지

**"Lessons Learned" (교훈) 신호:**
- 교훈, 배운 점, 주의, 실패, 삽질, 피해야, 하지 말, gotcha, pitfall, lesson
- ~가 문제였다, ~할 때 주의, 다시는, 이번에 깨달은 것

주제 키워드와 이 신호 단어가 **같은 섹션**(±10줄 이내)에 같이 나타나는 경우를 우선 추출한다.

### Step 4: 리포트 작성

아래 형식으로 출력한다.

```markdown
# 🔍 Find Context Report

**주제**: {주제}
**스캔한 문서**: N개 / **매치 섹션**: M개

## 📌 의사결정 (Why)

### 1. {결정 항목 요약}
- **출처**: `docs/prd/checkout/payment.md:42`
- **결정**: {무엇을 결정했는지 — 한 줄}
- **이유**: {why — 한두 줄}

### 2. ...

## 💡 교훈 (Lessons Learned)

### 1. {교훈 요약}
- **출처**: `.omniscitus/2026-W12.md:15`
- **내용**: {교훈}
- **맥락**: {어떤 상황에서 얻은 교훈인지}

## 🔗 추가로 읽어볼 만한 문서

- `path:L123` — {한 줄 설명}
- ...

---

💬 **다음 행동 제안**:
이 맥락을 반영해서 {현재 대화 중인 작업}을 진행할까요?
아니면 더 좁혀서 찾고 싶은 부분이 있나요?
```

### 결과가 없을 때

- 전혀 매치되는 문서가 없으면:
  ```
  🔍 관련된 과거 맥락을 찾지 못했습니다.

  이 주제는 이번이 처음인 것 같아요.
  작업을 진행하면서 결정 사항과 교훈이 쌓이면,
  /5-detail-prd 나 wrap-up 시 기록해두면 다음에 /find-context로 찾을 수 있어요.
  ```
- 매치는 있지만 "why"/"lessons" 신호가 약하면 **"참고 문서"** 섹션만 채워서 그 사실을 명시.

## 원칙

- **재사용성 우선**: 이 skill은 문서를 요약·인용만 한다. 문서를 수정하지 않는다.
- **출처 필수**: 모든 항목은 `file:line` 출처를 포함 (사용자가 직접 원문을 확인할 수 있도록).
- **추측 금지**: 문서에 없는 이유를 만들어내지 않는다. 신호가 약하면 "명시되지 않음"이라고 표시.
- **1가설 1검증과는 다르다**: 이건 탐색/리포트 도구이므로 여러 문서를 병렬로 스캔해도 됨.
