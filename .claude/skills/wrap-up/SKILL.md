# Wrap Up

세션 종료 시 작업 내용을 도메인별로 정리하여 기록하는 skill입니다.

## When to Use

- 사용자가 `/wrap-up` 을 입력했을 때
- 사용자가 "정리해줘", "마무리", "wrap up" 등을 요청할 때

## Instructions

### Step 1: 세션 분석

현재 세션에서 수행한 작업들을 분석합니다.

1. git diff로 변경사항 확인 (변경된 파일이 있는 경우)
2. 대화 내용에서 수행한 작업 목록 정리
3. 각 작업이 어떤 도메인에 해당하는지 분류

### Step 2: 도메인 분류

작업들을 도메인별로 분류합니다.

**기본 도메인:**

| 도메인 | 설명 |
|--------|------|
| `server` | 백엔드, API, DB, 인증 관련 |
| `web` | 프론트엔드, UI, 컴포넌트, 스타일링 |
| `native` | 모바일, 데스크톱 앱 관련 |
| `devops` | CI/CD, 배포, 인프라, 환경 설정 |
| `product` | 기획, PRD, 요구사항, 프로토타입 |

**새 도메인이 필요한 경우:**
1. 기존 도메인에 해당하지 않는 작업이 있으면 AskUserQuestion으로 제안:
   - question: "{작업내용}은 기존 도메인에 안 맞는데, 새 도메인을 만들까요?"
   - options에 2~3개 후보 제안 + "직접 입력" 옵션
2. 현재 `.claude/wrap-up/` 하위 폴더 수를 확인하여 **20개를 초과하지 않도록** 제어
3. 20개에 근접하면 기존 도메인에 병합할 수 있는지 우선 검토

### Step 3: wrap-up 문서 생성

각 도메인별로 문서를 생성합니다.

**경로**: `.claude/wrap-up/{domain}/{yyyy-mm-dd-shorten-title}.md`
- `{domain}`: 소문자, 하이픈 없이
- `{yyyy-mm-dd}`: 오늘 날짜
- `{shorten-title}`: 핵심 내용 2~4단어 kebab-case (예: `api-auth-setup`, `landing-page`)

**문서 형식**:

```markdown
# {제목}

## 요약
{이번 세션에서 해당 도메인에서 수행한 작업을 2~3줄로}

## 배경
{왜 이 작업을 했는지, 어떤 맥락에서 진행했는지}

## 배운점
{새롭게 알게 된 것, 삽질했던 것, 기억할 만한 것}

## 후속 작업
- [ ] {아직 안 한 것, 다음에 해야 할 것}

## 그외
{특이사항, 참고 링크, 메모 등. 없으면 "없음"}
```

**규칙:**
- 5개 섹션(요약/배경/배운점/후속작업/그외)만 사용. 추가하지 않는다
- 각 섹션은 간략하게. 요약은 3줄 이내, 배운점도 핵심만
- 이번 세션에서 N개 도메인에 걸쳐 작업했으면 N개 파일을 각각 생성

### Step 4: 폴더 생성 + 파일 저장

```bash
mkdir -p .claude/wrap-up/{domain}
```

Write 도구로 파일을 생성합니다.

### Step 5: 결과 출력

```
✅ Wrap-up 완료!

📝 저장된 기록:
  1. .claude/wrap-up/{domain1}/{date}-{title}.md
  2. .claude/wrap-up/{domain2}/{date}-{title}.md
  ...

📊 도메인 현황: {현재 도메인 수}/20
```
