# Follow Up

wrap-up으로 기록된 문서들을 확인하고, 후속 작업 상태를 점검하는 skill입니다.

## When to Use

- 사용자가 `/follow-up` 을 입력했을 때
- 사용자가 "f/u", "팔로업", "후속 작업 확인", "지난 작업 확인" 등을 요청할 때

## Instructions

### Step 1: history 디렉토리 구조 확인

`history/` 하위의 도메인 폴더와 문서 목록을 파악합니다.

```bash
find history/ -name "*.md" -type f | sort -r
```

- 도메인별로 어떤 문서가 있는지 정리
- 각 문서의 날짜(파일명 앞 `yyyy-mm-dd`)를 기준으로 최신순 정렬
- history/ 폴더가 없거나 비어있으면 안내 후 종료:
  ```
  📭 아직 기록된 wrap-up 문서가 없습니다. `/wrap-up`으로 먼저 작업을 기록해주세요.
  ```

### Step 2: 확인 범위 결정

AskUserQuestion으로 사용자에게 확인 범위를 묻습니다.

- **question**: 아래 형식으로 현황을 보여주며 질문:
  ```
  📋 history 현황:
    - {도메인1}: {N}개 문서 (최신: {날짜})
    - {도메인2}: {N}개 문서 (최신: {날짜})
    ...

  어느 범위까지 확인할까요?
  ```
- **options**:
  1. `최근 3일` (default) — 가장 최신 문서 날짜 기준 3일 이내
  2. `최근 7일` — 가장 최신 문서 날짜 기준 7일 이내
  3. `전체` — 기록된 모든 문서
  4. `직접 설정` — 날짜 범위나 특정 도메인 지정

**"직접 설정" 선택 시:** 추가 AskUserQuestion으로 구체적 범위를 받습니다.
- 예시: "server 도메인만", "3월 20일 이후", "최근 5개 문서" 등

### Step 3: 대상 문서 읽기 및 후속 작업 수집

Step 2에서 결정된 범위에 해당하는 wrap-up 문서들을 Read 도구로 읽습니다.

각 문서에서 수집할 정보:
1. **요약** — 무슨 작업이었는지
2. **후속 작업** (`## 후속 작업` 섹션) — 체크박스 항목들
3. **배운점** — 이어서 작업할 때 참고할 내용

### Step 4: 후속 작업 상태 점검

수집된 후속 작업 항목들의 현재 상태를 확인합니다.

1. **코드/파일 기반 확인**: 후속 작업에 언급된 파일, 기능, 설정 등이 현재 코드베이스에 반영되었는지 확인
   - Glob, Grep, Read 도구로 실제 코드 상태 확인
   - git log로 관련 커밋이 있는지 확인
2. **상태 분류**:
   - ✅ 완료 — 코드에 반영됨
   - 🔄 진행중 — 부분적으로 반영됨
   - ⬜ 미착수 — 아직 반영 안 됨
   - ❓ 확인 불가 — 코드로 판단하기 어려움 (사용자에게 확인 필요)

### Step 5: 결과 출력

```
📊 Follow-up 결과

━━━ {도메인1} ━━━

📄 {날짜} {제목}
  ✅ {완료된 항목}
  ⬜ {미착수 항목}
  🔄 {진행중 항목}

📄 {날짜} {제목}
  ...

━━━ {도메인2} ━━━
  ...

━━━━━━━━━━━━━━━━━

📈 전체 현황: ✅ {N}개 완료 / 🔄 {N}개 진행중 / ⬜ {N}개 미착수
🎯 우선 추천: {가장 시급하거나 의존성이 높은 미착수 항목 1~2개}
```

### Step 6: wrap-up 문서 업데이트

확인 결과를 바탕으로 원본 wrap-up 문서의 `## 후속 작업` 체크박스를 업데이트합니다.

- `- [ ]` → `- [x]` (완료 확인된 항목)
- Edit 도구로 해당 문서를 직접 수정
- 변경한 문서 목록을 마지막에 안내
